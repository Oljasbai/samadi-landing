# 🚀 ПОШАГОВАЯ ИНСТРУКЦИЯ ДЕПЛОЯ НА NETLIFY

## 📦 СПОСОБ 1: ПРЯМАЯ ЗАГРУЗКА (2 МИНУТЫ)

### Шаг 1: Скачайте все файлы проекта

1. В текущем интерфейсе Genspark найдите кнопку **"Download"** или **"Скачать проект"**
2. Скачается ZIP-архив с файлами
3. **Распакуйте** архив в любую папку

**Структура должна быть:**
```
samadi-landing/
├── index.html
├── css/
│   └── style.css
├── js/
│   └── script.js
├── README.md
└── другие файлы...
```

---

### Шаг 2: Откройте Netlify

1. Перейдите на **https://www.netlify.com**
2. Нажмите кнопку **"Sign up"** (Регистрация) в правом верхнем углу

**Или сразу авторизуйтесь через GitHub** (рекомендуется для интеграции):
- Нажмите **"Sign up with GitHub"**
- Разрешите доступ Netlify к GitHub
- ✅ Готово - вы авторизованы!

---

### Шаг 3: Перетащите папку

1. После входа вы увидите главную страницу Netlify
2. Найдите большую область с текстом:
   **"Want to deploy a new site without connecting to Git?**
   **Drag and drop your site output folder here"**

3. **Откройте папку** с проектом (распакованную на Шаге 1)
4. **Перетащите ВСЮ ПАПКУ** `samadi-landing` в эту область
   
   💡 **Важно:** Перетаскивайте папку целиком, а не отдельные файлы!

5. Netlify начнет загрузку (10-20 секунд):
   - Вы увидите прогресс загрузки
   - Затем автоматический деплой

---

### Шаг 4: Получите ссылку

1. После деплоя (20-30 секунд) появится сообщение:
   **"Your site is deployed"**

2. Вы увидите **URL вашего сайта**:
   ```
   https://random-name-123456.netlify.app
   ```

3. **Скопируйте эту ссылку** - это адрес вашего сайта!

4. **Откройте её в браузере** - сайт работает! ✨

---

### Шаг 5: Измените название (опционально)

1. На странице сайта в Netlify нажмите **"Site settings"**
2. Найдите раздел **"Site information"**
3. Нажмите **"Change site name"**
4. Введите, например: **samadi-kz**
5. Сохраните

Теперь ваш сайт: **https://samadi-kz.netlify.app** 🎉

## 🔗 СПОСОБ 2: ИНТЕГРАЦИЯ С GITHUB (Рекомендуется)

### Преимущества GitHub интеграции:
- ✅ Автоматический деплой при каждом изменении
- ✅ История всех версий сайта
- ✅ Легко откатиться к предыдущей версии
- ✅ Командная работа
- ✅ Профессиональный подход

---

### Шаг 1: Создайте репозиторий на GitHub

1. **Перейдите на** https://github.com
2. **Войдите** в свой аккаунт (или создайте новый)
3. Нажмите **зеленую кнопку "New"** или **"New repository"**

4. **Заполните форму:**
   - **Repository name:** `samadi-landing`
   - **Description:** "Корпоративный лендинг компании САМАДИ"
   - **Visibility:** Public (можно изменить на Private)
   - ✅ Поставьте галочку **"Add a README file"**
   - Нажмите **"Create repository"**

5. ✅ Репозиторий создан!

---

### Шаг 2: Загрузите файлы в GitHub

#### Вариант 2.1: Через веб-интерфейс (Простой)

1. В вашем репозитории нажмите **"Add file"** → **"Upload files"**

2. **Перетащите все файлы** проекта в окно:
   - `index.html`
   - Папку `css/` (с файлом style.css внутри)
   - Папку `js/` (с файлом script.js внутри)
   - `README.md`
   - Все остальные файлы

3. В поле **"Commit changes"** внизу напишите:
   ```
   Initial commit - САМАДИ лендинг
   ```

4. Нажмите **"Commit changes"**

5. ✅ Файлы загружены!

---

#### Вариант 2.2: Через командную строку (Продвинутый)

**Если у вас установлен Git:**

```bash
# 1. Откройте терминал в папке проекта
cd путь/к/samadi-landing

# 2. Инициализируйте Git
git init

# 3. Добавьте все файлы
git add .

# 4. Сделайте первый коммит
git commit -m "Initial commit - САМАДИ лендинг"

# 5. Подключите к GitHub (замените YOUR-USERNAME на ваш логин)
git remote add origin https://github.com/YOUR-USERNAME/samadi-landing.git

# 6. Отправьте код на GitHub
git branch -M main
git push -u origin main
```

**Готово!** Код на GitHub ✅

---

### Шаг 3: Подключите Netlify к GitHub

1. **Перейдите на** https://app.netlify.com

2. **Нажмите** кнопку **"Add new site"** → **"Import an existing project"**

3. **Выберите** "Deploy with GitHub"

4. **Разрешите доступ** Netlify к GitHub (если спросит)

5. **Выберите репозиторий** `samadi-landing` из списка

6. **Настройки деплоя:**
   - **Branch to deploy:** `main`
   - **Build command:** оставьте пустым
   - **Publish directory:** оставьте пустым

7. **Нажмите** "Deploy site"

8. **Ждите** 30-60 секунд... 

9. ✅ **Готово!** Сайт опубликован!

---

### Шаг 4: Получите ссылку

После деплоя вы увидите:
```
✅ Site is live
https://random-name-123456.netlify.app
```

**Скопируйте и откройте** эту ссылку - сайт работает!

## 🔧 ДОПОЛНИТЕЛЬНЫЕ НАСТРОЙКИ

### 1. Изменить имя сайта

**В Netlify:**
1. Site settings → Site details
2. Change site name
3. Введите: `samadi-kz`
4. Save

**Новый URL:** `https://samadi-kz.netlify.app`

---

### 2. Подключить свой домен

**Если у вас есть домен (samadi.kz):**

1. **В Netlify:**
   - Site settings → Domain management
   - Add custom domain
   - Введите: `samadi.kz`

2. **У вашего регистратора доменов:**
   - Добавьте DNS запись:
   ```
   Type: A
   Name: @
   Value: 75.2.60.5
   ```
   ```
   Type: CNAME
   Name: www
   Value: samadi-kz.netlify.app
   ```

3. **Подождите** 24-48 часов распространения DNS

4. ✅ **Готово!** Сайт доступен на `https://samadi.kz`

**Bonus:** Netlify автоматически добавит бесплатный SSL сертификат!

---

### 3. Настроить защиту паролем (опционально)

**Если сайт ещё не готов к публике:**

1. Site settings → Access control
2. Enable "Password protection"
3. Установите пароль
4. Теперь сайт доступен только с паролем

## 🆘 РЕШЕНИЕ ПРОБЛЕМ

### ❌ "Сайт не загружается"

**Проверьте:**
1. Правильно ли загружены файлы?
   - В Netlify: Deploys → последний deploy → Deploy log
   - Должно быть "Deploy succeeded"

2. Все ли файлы на месте?
   - В GitHub проверьте структуру
   - Должны быть: index.html, css/style.css, js/script.js

3. Очистите кэш браузера:
   - Windows/Linux: `Ctrl + Shift + R`
   - Mac: `Cmd + Shift + R`

---

### ❌ "Стили не применяются"

**Если на Netlify всё равно нет стилей:**

1. Откройте Developer Tools (F12)
2. Вкладка "Console"
3. Посмотрите на ошибки

**Скорее всего:**
- Файл `css/style.css` не загружен
- **Решение:** Проверьте, что папка `css` с файлом `style.css` есть в репозитории

---

### ❌ "GitHub спрашивает логин и пароль"

**При push через командную строку:**

1. **Создайте Personal Access Token:**
   - GitHub → Settings → Developer settings → Personal access tokens
   - Generate new token (classic)
   - Выберите срок действия
   - Поставьте галочку "repo"
   - Generate token
   - **СКОПИРУЙТЕ TOKEN** (больше не увидите!)

2. **При запросе пароля** вставьте TOKEN вместо пароля

---

### ❌ "Netlify показывает 404"

**Проверьте:**
- Файл `index.html` находится в корне проекта?
- Название файла правильное? (не `Index.html` или `INDEX.html`)

**Исправление:**
- Переименуйте файл в `index.html` (маленькие буквы)
- Закоммитьте изменения
